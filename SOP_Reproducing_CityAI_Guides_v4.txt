SOP_Reproducing_CityAI_Guides_v4
=========================================================================
Operational Procedure — City-AI Project
Supersedes v3. Aligns with Governance v7: MDR is sole authority for variables.

0. Purpose
- Define the workflow for generating, auditing, and archiving guides under Governance v7.
- Ensure MDR (variables) and Intake (assets/systems) remain synchronized.

1. Before You Start a Guide
- Load the following authoritative files:
  • CityAI_Governance_MasterRules_v7.txt
  • CityAI_Project_50_Guides_Outline.txt
  • Latest Intake SSOT (UE561_CityAI_MasterLog_FULL_v*.txt)
  • Latest MDR (CityAI_MasterDeclarationRegistry_v*.txt)
- Confirm the target scope from the Outline.

2. Authoring a Guide (Exhaustive Drafting)
- Use Teacher Format sections: Intro Narrative, Declarations Introduced, Background Concepts,
  Step-by-Step, Verification Checklist, Common Mistakes and Fixes, Big Picture Connection.
- Apply UE5.6.1 Standards (Enhanced Input, variable standards, tags, categories, editable defaults).
- Declare both immediate and foreseeable needs (1–2 guides ahead).
- Save deliverable as FULL .txt.

3. Recording Declarations
- For every new **asset, system, or function** introduced:
  • Add it to the Intake staging list as [EXPANSION].
- For every new **variable** introduced:
  • Add it to the MDR with:
    – Name, type, defaults, metadata, tags.
    – Status ([BASE] or [EXPANSION]).
    – Guide of Origin (Guide X).
- Example Intake entry:
  BP_VehicleBase (Pawn) [EXPANSION]
    • Variables: See MDR v1 (Guide 1)
- Example MDR entry:
  BP_VehicleBase.MaxSpeed (Float, default=1200, Category=Movement, Editable=True) [BASE] — Guide 1

4. Intake and MDR Updates
- At every 10-guide checkpoint:
  • Merge expansions into new Intake version (assets/systems only).
  • Merge variables into new MDR version.
  • Increment both filenames (vN → vN+1).

5. Audit at 10-Guide Checkpoint
- Produce:
  • New Intake version.
  • New MDR version.
  • Audit Report verifying:
    – UE5.6.1 compliance.
    – Outline alignment.
    – Intake vs MDR synchronization (no variable duplication in Intake).
    – Guide-of-Origin tagging accuracy.

6. Storage and Versioning
- Filenames:
  • Intake: UE561_CityAI_MasterLog_FULL_v{N}.txt
  • Governance: CityAI_Governance_MasterRules_v{N}.txt
  • SOP: SOP_Reproducing_CityAI_Guides_v{N}.txt
  • MDR: CityAI_MasterDeclarationRegistry_v{N}.txt
  • Audits: Audit_Report_v{N}.txt
- Old versions archived, never overwritten.
