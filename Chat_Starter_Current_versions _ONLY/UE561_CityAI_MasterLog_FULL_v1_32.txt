============================================================
SECTION 1) PROJECT GOALS AND SCOPE

Primary outcome:
- Living city simulation with AI vehicles and pedestrians.
- Possession-capable for vehicles and pedestrians.
- Random GPS-like guidance including optional sky-road vehicles.
- Deterministic PCG with fixed seeds for reproducible layouts.

Scope:
- Mid-sized downtown with suburbs.
- Incremental build from 1 car + 1 pedestrian to dense city.
- US traffic rules and guidelines. Signals, stops, yields.

Non-goals:
- No external DCC assets. In-editor modeling only.
- No multiplayer. Single-player only.
- No economy simulation.

Player interaction:
- Spectate or possess any car or pedestrian at will.

World:
- Single large map. World Partition enabled.


============================================================
SECTION 2) PLATFORMS AND PERFORMANCE BUDGETS

Platform:
- Windows PC.

Hardware target (baseline):
- CPU: Intel Core i7-4790 (4C/8T).
- RAM: 32 GB.
- GPU: AMD Radeon RX 6900 XT, 8 GB VRAM.

Performance:
- 50 FPS minimum average in city-scale scenes.

Streaming:
- World Partition, cell sizing tuned via testing.
- HLOD aggressive at distance.

Concurrency targets (visible at once; pool outside relevance):
- Vehicles: ~300 citywide.
- Pedestrians: ~600 citywide.


============================================================
SECTION 5) AGENT TYPES AND BEHAVIOR

Vehicles:
- Sedans, SUVs, Trucks, Buses, Motorcycles, Taxis, Emergency (Police/Ambulance/Fire), Service, SkyVehicle.

Pedestrians:
- Commuter, Shopper, Jogger, Tourist, Vendor.

Behavior profiles:
- Sedans/SUVs: compliant/cautious with minor distraction.
- Buses/Trucks: cautious, wider turns, longer braking.
- Motorcycles: aggressive, fast lane changes.
- Taxis: aggressive routing, risky merges.
- Emergency: aggressive when active with sirens.
- Service: frequent stops.

Pedestrian obedience:
- 85% obey signals.
- 15% jaywalk on safe gaps.

Ownership:
- Pedestrians can enter vehicles and become drivers.
- Not all pedestrians drive. Sidewalk activity preserved.
- Driver proxying allowed for performance while keeping sidewalks populated.

Emergency:
- Respond to crashes. Spawn from appropriate garages.
- After response, return to garage and despawn inside.


============================================================
SECTION 6) NAVIGATION AND PATHFINDING

Vehicles:
- Hybrid: global A* on lane graph + local spline following.
- Mass Traffic processors for scalability.

Pedestrians:
- NavMesh with off-mesh links for crosswalks.
- Mass Crowd used for scaling and local avoidance.

Graph data:
- PCG outputs splines and creates node/edge graph for routing.

Route choice:
- Weighted by time-of-day schedules and destination types.
- Congestion-aware weights from analytics heatmaps.

Stuck handling:
- Attempt reroute twice using local detours.
- If still blocked, compute a new route from current node.

Sensors:
- Vehicles use raycasts + sphere overlaps.
- Pedestrians perceive vehicles for avoidance and crosswalk safety.


============================================================
SECTION 7) TIME, WEATHER, LIGHTING

Time of day:
- Dynamic day/night with adjustable time scale.

Lighting:
- Lumen enabled. Nanite for meshes.

Weather:
- Clear, Overcast, Fog, LightRain, HeavyRain.
- Weather affects visibility and traction.
- Night + rain combine to reduce sensor confidence; vehicle speeds adapt.


============================================================
SECTION 8) DEBUGGING AND ANALYTICS

Master debug toggle:
- Enables/disables all debug overlays and logs.

Overlays:
- Traffic density heatmap.
- Pedestrian density.
- Path tracer (recent AI paths).
- Sensor rays and perception.
- Personality diagnostics.
- Event log timeline.
- Replay controller.

Exports:
- CSV logs for QA runs.


============================================================
SECTION 9) PERSISTENCE

Save/Load:
- Saves seed, time, weather, player position, settings, debug toggles.
- Agents do not persist between sessions; always reset on load.
- Focus is on reproducibility and configuration.


============================================================
SECTION 10) UI AND MENUS

HUD and UI:
- Minimap with GPS routing and traffic flow overlay.
- Main menu and settings (video/audio/accessibility).
- Taxi request panel with ETA.
- Garage menu for owned/unlocked vehicles (no economy).
- Notifications panel.
- Tutorial panel for onboarding.
- Help panel with searchable dev docs.
- Save slot UI.
